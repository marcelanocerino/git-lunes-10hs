
FETCH => trae el historial de cambios al repositorio pero NO MODIFICA el WORKING DIRECTORY
	 (Solamente Actualiza la carpeta .git)

PULL  => trae el historial de cambios y combina los mismos con nuestro WORKING DIRECTORY generando un nuevo Commit
	 (Realiza un FETCH + MERGE)

PUSH  => envia el historial de cambios y los archivos al REPOSITORIO REMOTO
	 (Permite crear/eliminar ramas de los remotos)
