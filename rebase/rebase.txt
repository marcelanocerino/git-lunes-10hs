REBASE
	Herramienta que perite reorganizar los commits de una rama,
	integrar los commits de 2 ramas de manera organizada, o
	reaplicar los commits en una rama desde el commit especificado.
	
	NOTA: No debemos aplicar este comando si los cambios ya fueron
	subidos a nuestro repositorio remoto, ya que altera el historial
	de commits implicados.

GIT
	git rebase => reorganiza los commits pertenecientes a la rama (que no afecten a otras)
	git rebase <branch> => reorganiza los commits de ambas ramas cronologicamente
	git rebase <commit> => reorganiza el historial a partir del commit indicado
	
	git rebase -i <param> => abre el editor para indicar como aplicar los commits (editar historial)
